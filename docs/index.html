WiFi Dashboard

Real-time WiFi network scanner, channel analyzer, and signal monitor

Quick Start Features API Config Testing Structure

Quick Start

3 commands to start scanning

git clone <repo-url> wifi-dashboard
cd wifi-dashboard
bash launch.sh start

Opens http://localhost:8002 automatically. Click Scan or Demo to begin.

Manual Start

python -m venv .venv
source .venv/bin/activate
pip install -r backend/requirements.txt
python backend/main.py

Features

Scanner

Discover 2.4 and 5 GHz networks. Filter by SSID, signal strength, or band. Export to CSV or HTML.

Channel Analyzer

Visual 2.4/5 GHz congestion maps. Overlap scoring. Best-channel recommendations for your router.

Signal Monitor

Live RSSI chart for any network. Min/Max/Avg stats. Adjustable history window. Data export.

7 Radar Styles

Military, Tactical, Thermal, Sonar, Stars, Matrix, Submarine. Canvas-animated with signal-to-distance mapping.

Achievements

20 badges, trading cards, missions, XP and levels. Gamified network discovery.

Hacker Lab

Network autopsy, packet storm, time travel scanner, RF waterfall, terminal view.

Command Center

Analytics, vendor charts, RSSI histogram, leaderboard, heatmap, co-location graph, stealth meter.

Vendor Lookup

260+ OUI prefixes. Identify TP-Link, Netgear, Apple, Ubiquiti, Cisco, and more by MAC address.

Themes

8 visual themes: Mosque, Zellige, Andalus, Riad, Medina, Space, Jungle, Robot

Languages

English, French, Arabic (RTL support)

API Reference

WS WebSocket /ws

Connect and send JSON messages for real-time scanning and analysis.

SendReceiveDescription
{"type":"hello"}hello_ackServer handshake
{"type":"scan", "simulate":true}scan_resultScan for networks
{"type":"scan_stop"}—Stop active scan
{"type":"channel_analysis"}channel_analysisFull congestion data
{"type":"best_channel", "band":"2.4GHz"}best_channelBest channel + reason

Scan Parameters

FieldTypeDefaultDescription
durationfloat5.0Scan duration in seconds
ssid_filterstring""Filter by SSID substring
rssi_minint-100Minimum signal strength
bandstring"all""all", "2.4GHz", or "5GHz"
simulateboolfalseUse simulated data
